Creative direction map

Choose from the whole bench.

The landing page should preview the operating model: start from the files that exist, pull in specialist workflows only when they sharpen the artifact, and keep external services optional until a concrete asset is needed.

Design files Canonical page first.

index.html stays the public source of truth; README.md anchors the repo purpose without inflating the story.

Skills Direction before decoration.

Creative direction and design review are the useful lane here; media, deck, and ad skills wait until the page needs those surfaces.

Templates Optional export path.

Screenshot, accessibility, PDF, or hosting templates are useful for review and publishing, not for inventing new page content.

Connectors No dependency yet.

With no connected sources, the honest fallback is a static, inspectable page that can later point at real project receipts.

Design posture

Useful, then public.

The page should make one promise: Hermegeddon publishes work only when it has enough context, source, and maintenance surface to be inspected later.
